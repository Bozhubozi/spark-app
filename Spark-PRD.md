## 1. 产品概述

### 1.1 产品定位

**一句话定位：Soul 是游乐场，Spark 是搭子搜索引擎。**

Spark 是一款面向 18-28 岁年轻人的社交 App，核心理念是**做减法**——不建社区、不做 Feed、不做语音房，只做一件事：**用最短路径帮你找到对的人，然后去线下。**

与 Soul 的关键差异：

| 维度 | Soul | Spark |
|---|---|---|
| 核心玩法 | 星球匹配、语音速配、群聊派对——偏娱乐消遣 | 人格+兴趣+星座+LBS 四维匹配——偏效率连接 |
| 使用动机 | 无聊时打开逛逛，kill time | 想找人一起做某件事，solve need |
| 关系形态 | 广泛社交、群聊、Feed——重社区 | 1 对 1 场景化搭子——重连接质量 |
| 匹配方式 | 星球旋转、语音匹配——随机性强 | 四维算法 + LBS——精准度高 |
| AI 角色 | AI 虚拟伙伴陪你聊 | AI 帮你找到对的人（人格报告/破冰/星座适配） |
| 功能密度 | 极重——语音房、Feed、群组、游戏化 | **极简——只保留匹配+聊天+AI 辅助** |
| 视觉风格 | 暖色、卡通、元宇宙风 | 暗色、简约、霓虹风 |

**Soul 有的，Spark 不做：** 语音房/群聊派对、朋友圈/动态 Feed、视频聊天、复杂形象拼装、等级积分体系、AI 虚拟伙伴陪聊

**Spark 做的，Soul 不够：** 人格+兴趣+星座+LBS 四维匹配算法、日推 30 人克制设计（刷完即走）、AI 破冰话题消除沉默、同城搭子导向线下、AI 人格报告可分享获客

### 1.2 对标产品

| 竞品 | Spark 差异 |
|---|---|
| **Soul** | Soul 是"社交游乐场"（随机匹配+语音+群聊+社区+AI陪伴），Spark 是"搭子搜索引擎"（精准匹配→破冰→线下），功能密度约为 Soul 的 1/5 |
| **探探** | 探探以真实照片滑动为主，Spark 纯虚拟形象+数据卡片，不看脸看数据 |
| **积目** | 积目偏小众潮流社区，Spark 不做社区，专注 1 对 1 连接 |

### 1.3 北极星指标

**周活跃匹配对数**（matched pairs per week）

### 2.3 用户场景

**场景 1（画像 1：二次元玩家）**：大二学生小林晚上刷 B 站看到原神新角色 PV，想找个也玩原神的搭子但身边同学都只打王者。打开 Spark，看到卡片上兴趣标签写着"原神""谷子""K-pop"，星座天蝎，人格"开放性"高分——这是她要找的人。右滑匹配，系统弹出星座适配报告"天蝎 × 双鱼，适配度 87%"。用虚拟形象发送第一条消息："你也抽了芙宁娜？"

**场景 2（画像 2：手工疗愈型）**：26 岁上班族小陈周末拼完一副拼豆，拍完发小红书后有点孤单。打开 Spark，人格测试做完，系统推的人"都对味"。匹配到同样喜欢攀岩和脱口秀的人，破冰话题自动生成——"你们岩馆周末人多吗？"——算法发现两人兴趣高度重合。

**场景 3（画像 4：线下连接者）**：24 岁设计师阿杰发现 Instagram 上 #HobbyMaxxing 很火，想加个本地跑步俱乐部但不知道去哪找。Spark 上筛选"同城 + 跑步"标签，匹配到一个也跑马拉松的人。系统提示："你们都在朝阳区，距离 2km"。两人约了周末奥森跑步——不是因为想谈恋爱，而是"缺个跑步搭子"。这也是 Spark 要承载的关系形态。

**场景 4（画像 5：自我探索者）**：19 岁大一新生小鹿刚做完 MBTI 测试（INFP），还想更了解自己。完成 Spark 10 题大五人格，得到一份 AI 人格报告："高开放性 + 高神经质——你适合找情绪稳定的宜人型朋友"。系统推送匹配时标注了"人格互补度 92%"，小鹿觉得"比我自己还懂我"。

## 4. 核心流程

### 4.1 新用户引导流程

```
下载 App → 进入发现页（无需注册，直接开刷）
  ├── 浏览匹配卡片（限 5 张预览，仅展示脱敏数据：虚拟形象 + 星座 + 人格维度）
  ├── 查看星座运势入口（预览，点击触发注册）
  └── 人格测试入口（预览，点击触发注册）

触发注册（右滑 like / 发消息 / 查看星座适配 / 查看人格报告）
  → 微信一键登录 或 手机号+验证码
  → 身份证+人脸实名验证

注册后 → 返回之前触发的操作（不打断用户意图）
  → 选择兴趣标签（至少 5 个，必做）→ 立即提升匹配精准度
  → 完成人格测试（可跳过，完成后解锁 AI 人格报告）
  → 选择虚拟形象风格（可跳过，预设默认形象）
  → 主页（发现页，匹配质量已提升）
```

> **合规说明**：预览模式仅展示脱敏数据（虚拟形象、星座、人格维度），不展示昵称、距离、Bio 等可定位到具体个人的信息。正式开发前需法务确认"先体验后注册"模式符合当前监管要求。

### 4.2 匹配流程

```
系统推送候选池（日限 100 人，前端展示 30 张卡片）
  ├── 候选池逻辑：后端每次计算 100 人候选池，前端每次抽取 30 张卡片展示
  ├── 用户当天刷完 30 张即止（"刷完即走"），次日重新生成新候选池
  ├── 卡片展示：兴趣标签 + 星座 + 人格维度为主，虚拟形象缩小
  ├── 支持筛选：星座 / 兴趣 / 同城（LBS）
  └── 排序算法：人格匹配度 + 兴趣 Jaccard + 星座适配 + 活跃度 + LBS 距离

用户左滑（pass）或右滑（like）
  → A 右滑 B → 状态 pending
  → B 右滑 A → 双向匹配
       → WebSocket 实时推送 + JPush 离线通知
       → 全屏匹配弹窗（含 AI 星座适配报告）
       → 自动创建聊天室 + AI 生成破冰话题
  → B 左滑 A → A 的 pending 记录作废
```

### 4.3 聊天流程

```
匹配成功 → 自动创建聊天室
  → 系统推送 AI 破冰话题（基于共同兴趣 + 星座自动生成）
  → 双方收到 match.new WebSocket 推送

聊天中：
  → WebSocket 实时收发文字消息 + 基础 Emoji
  → client_msg_id 幂等去重（数据库唯一索引）
  → 发送状态：发送中 → 已送达 → 已读（V1.1）
  → 接收方在线 → 实时推送 chat.message.new
  → 接收方离线 → 消息入 Redis 队列（7 天 TTL）→ 重连后自动同步
  → 长按消息：拉黑/举报入口（带原因选择）
```

## 6. 约束与风险

| 约束/风险 | 说明 | 应对 |
|---|---|---|
| 冷启动 | 初期用户少，匹配密度低 | 1) 种子用户 100 人（5 画像 × 20 人、5 城市 × 20 人，W6-W8 招募，W11 入驻），渠道：小红书 KOC + B 站 UP 主 + 员工校友推荐。2) 虚拟 Bot 200 个（5 画像 × 40 个），仅作候选池填充（占比 ≤ 30%），被右滑后不响应。3) 候选池不足时自动放宽条件（距离 5→50km，标签 5→2 个）。4) 冷启动期（W12-W16）北极星指标暂定 30 对/周 |
| 实名制合规 | 18 周岁门槛，Soul 2019 年因未成年人问题被下架 | 身份证+人脸强实名验证，不做年龄自填 |
| 内容安全 | 聊天内容可能违规 | 敏感词过滤 + 举报机制 |
| iOS 审核 | 社交类 App 需 ICP 备案 + 资质 | 提前准备，V1.0 先上 TestFlight |
